# WarpAUV 强化学习控制工作的理论基础

本文档整理当前工作（基于 Isaac Lab 的推进器驱动 AUV 六自由度控制与轨迹跟踪）的理论基础。整体思路是：用水下机器人六自由度动力学描述被控对象，用推进器模型建立动作到体坐标系力/力矩的映射，再将定点保持与轨迹跟踪任务建模为马尔可夫决策过程，通过 PPO 学习从观测到推进器命令的闭环控制策略。

## 1. 研究对象与控制问题

本工作面向一类推进器驱动的自主水下航行器（Autonomous Underwater Vehicle, AUV）。AUV 在三维水体中运动，具有 6 个自由度：

- 平动自由度：surge、sway、heave，对应体坐标系线速度 `u, v, w`。
- 转动自由度：roll、pitch、yaw，对应体坐标系角速度 `p, q, r`。

通常定义位姿与速度为：

```text
eta = [x, y, z, attitude]
nu  = [u, v, w, p, q, r]^T
```

其中 `eta` 描述世界坐标系下的位置和姿态，`nu` 描述体坐标系下的线速度和角速度。本仓库中姿态主要使用四元数表示，以避免欧拉角奇异性；观测和目标姿态均保留四元数形式。

控制目标分为两类：

1. 定点/姿态保持：使 AUV 回到环境原点附近，并跟踪随机目标姿态，同时抑制线速度和角速度。
2. 轨迹跟踪：使 AUV 跟踪随时间变化的目标位置、姿态和目标线速度，包括训练阶段的平滑随机轨迹和评估阶段的外推轨迹。

## 2. 坐标系与姿态表示

水下机器人动力学通常区分世界坐标系和体坐标系：

- 世界坐标系用于描述全局位置、重力方向、水流方向和目标轨迹。
- 体坐标系固定在机器人本体上，用于描述本体速度、推进器布置、体坐标力和力矩。

二者之间通过旋转矩阵或四元数转换。若 `R(q)` 表示从体坐标系到世界坐标系的旋转，则：

```text
v_w = R(q) v_b
v_b = R(q)^T v_w
```

代码中使用 Isaac Lab 的四元数约定，并通过 `quat_apply` 和 `quat_conjugate` 完成坐标变换。例如轨迹跟踪观测中的目标位置误差被转换到体坐标系：

```text
e_p^b = R(q)^T (p_target^w - p_robot^w)
```

这样做的好处是策略输入具有更强的姿态不变性：同样的“目标在机器人前方”会被表示为类似的体坐标误差，而不是依赖世界坐标系朝向。

## 3. 六自由度水下机器人动力学

水下航行器常用 Fossen 风格的六自由度模型描述：

```text
eta_dot = J(eta) nu
M nu_dot + C(nu) nu + D(nu_r) nu_r + g(eta) = tau
```

其中：

- `M = M_RB + M_A`：刚体质量矩阵与附加质量矩阵。
- `C(nu) = C_RB(nu) + C_A(nu_r)`：刚体科氏/离心项与附加质量科氏项。
- `D(nu_r)`：水动力阻尼。
- `g(eta)`：重力与浮力引起的恢复力。
- `tau`：推进器等外部执行器产生的广义力。
- `nu_r = nu - nu_current`：相对水流速度。

本项目在 Isaac/PhysX 中由仿真器负责刚体惯性、刚体陀螺项和重力积分。因此 `rigid_body_hydrodynamics.py` 只额外施加流体相关外力，主要包括浮力、相对速度阻尼和可选的附加质量科氏项。这个边界很重要，可以避免重复计算刚体动力学和重力。

## 4. 浮力与恢复力矩

浮力大小由阿基米德原理给出：

```text
F_b^w = - rho V g^w
```

其中 `rho` 是水密度，`V` 是排水体积，`g^w` 是世界坐标系重力向量。负号表示浮力方向与重力相反。将浮力转换到体坐标系后：

```text
F_b^b = R(q)^T F_b^w
```

如果质心（COM）和浮心（COB）不重合，浮力会产生恢复力矩：

```text
tau_b^b = r_cb^b x F_b^b
```

其中 `r_cb^b` 是从质心到浮心的偏移。本项目默认设置浮心相对质心在 `z` 方向有小偏移，用于提供被动姿态恢复效应。若满足：

```text
m g ~= rho V g
```

则系统接近中性浮力，适合模拟多数水下机器人在工作深度附近的运动状态。

## 5. 水动力阻尼与相对流速

水下运动中，阻尼主要来自粘性阻力和形状阻力。本项目使用体坐标系下的对角线性阻尼与二次阻尼模型：

```text
D(nu_r) = D_l + D_q |nu_r|
tau_d = -D(nu_r) nu_r
```

其中 `nu_r` 是相对水体速度，而不是机器人绝对速度。这一点在有水流时尤其重要：水流可能对机器人做功，因此阻尼耗散性应针对相对速度判断：

```text
nu_r^T tau_d <= 0
```

代码中的 `tests/test_dynamics_math.py` 对这一耗散性质进行了检查。理论上，阻尼项提供速度衰减，帮助 AUV 在没有推进输入时逐渐稳定下来；控制策略则需要在阻尼、水流和目标运动之间学习合适的推进补偿。

## 6. 附加质量与科氏项

水下机器人加速时不仅需要加速自身质量，还会带动周围水体一起运动，这部分等效惯性称为附加质量。简化模型常使用对角附加质量矩阵：

```text
M_A = diag(X_dot_u, Y_dot_v, Z_dot_w, K_dot_p, M_dot_q, N_dot_r)
```

附加质量会引入附加科氏/离心项 `C_A(nu_r) nu_r`。合理构造的科氏项具有功率守恒性质：

```text
nu_r^T C_A(nu_r) nu_r = 0
```

也就是说，该项只改变动量方向，不直接耗散或注入能量。当前配置中 `added_mass_diag` 默认为 0，但代码已保留相应模型和数学测试，便于后续加入更真实的水动力参数。

## 7. 推进器动力学与力矩映射

AUV 的控制输入是 6 个推进器的归一化命令：

```text
a_i in [-1, 1], i = 1, ..., 6
```

仓库中动作先经过死区和 PWM 到电机转速的非线性映射：

```text
|a_i| < 0.08        -> omega_cmd = 0
a_i >= 0.08        -> omega_cmd = -139 a_i^2 + 500 a_i + 8.28
a_i <= -0.08       -> omega_cmd =  161 a_i^2 + 517.86 a_i - 5.72
```

随后使用一阶惯性描述推进器响应：

```text
omega_dot = (omega_cmd - omega) / tau
```

离散实现为指数平滑形式。该模型反映了电机/推进器不能瞬时达到目标转速的事实。

推进器产生的推力使用二次模型：

```text
T_i = k |omega_i| omega_i
```

该形式可以同时表示正反转推力方向，并体现推力近似与转速平方成正比。每个推进器在本体上的安装位置 `r_i` 和方向由外参给出，单个推进器产生：

```text
F_i^b = R_i [T_i, 0, 0]^T
tau_i^b = r_i x F_i^b
```

总推进力和总推进力矩为：

```text
F_thr^b = sum_i F_i^b
tau_thr^b = sum_i tau_i^b
```

最终施加到刚体上的外力/力矩为：

```text
tau_total = tau_thr + tau_fluid
```

其中 `tau_fluid` 包括浮力、阻尼和可选附加质量项。

## 8. 强化学习任务建模

本工作将控制问题表述为马尔可夫决策过程（MDP）：

```text
M = (S, A, P, R, gamma)
```

其中：

- `S`：系统状态，包括机器人位姿、速度、目标姿态、目标位置或目标轨迹信息。
- `A`：连续动作空间，即 6 个推进器命令。
- `P`：状态转移，由 Isaac/PhysX 刚体仿真、水动力模型和推进器模型共同决定。
- `R`：奖励函数，用于表达控制目标。
- `gamma`：折扣因子，用于平衡短期与长期收益。

策略网络学习一个从观测到动作的映射：

```text
a_t ~ pi_theta(a_t | o_t)
```

训练目标是最大化期望折扣回报：

```text
J(theta) = E[sum_t gamma^t r_t]
```

### 8.1 定点保持观测与奖励

定点保持任务的观测维度为 17，主要包括：

- 目标姿态四元数。
- 机器人相对环境原点的位置误差，表示在体坐标系中。
- 当前姿态四元数。
- 体坐标系线速度。
- 体坐标系角速度。

奖励由以下部分组成：

```text
r = r_pos + r_ang + r_lin_vel + r_ang_vel + r_action
```

其中：

```text
r_pos     = w_pos exp(-||e_p||^2)
r_ang     = w_ang exp(-quat_error(q_target, q))
r_lin_vel = w_lin exp(-||v_b||^2)
r_ang_vel = w_angvel exp(-||omega_b||^2)
r_action  = -w_action ||a||^2
```

该奖励结构属于典型的 shaped reward：位置和姿态越接近目标，奖励越高；速度项鼓励稳定悬停；动作惩罚抑制不必要的能耗和高频控制。

### 8.2 轨迹跟踪观测与奖励

轨迹跟踪任务的观测维度为 20，包含：

- 目标姿态四元数。
- 体坐标系下目标位置误差。
- 体坐标系下目标线速度。
- 当前姿态四元数。
- 当前体坐标线速度。
- 当前体坐标角速度。

奖励为：

```text
r = r_pos + r_ang + r_track_vel + r_ang_vel + r_action
```

其中位置奖励跟踪移动目标，速度奖励鼓励机器人速度匹配目标速度，而不是像定点保持任务那样将线速度压到 0：

```text
r_pos       = w_pos exp(-||p_target^w - p^w||^2)
r_track_vel = w_vel exp(-||v_target^b - v^b||^2)
```

因此，轨迹跟踪策略需要同时学习“追上目标”和“以目标速度运动”，避免只追求瞬时位置误差最小而产生滞后或超调。

## 9. PPO 算法基础

本仓库使用 RSL-RL 的 PPO（Proximal Policy Optimization）训练连续控制策略。PPO 是一种 on-policy actor-critic 方法，包含：

- Actor：输出动作分布 `pi_theta(a | o)`。
- Critic：估计状态价值 `V_phi(o)`。
- GAE：用广义优势估计降低方差。
- Clipped surrogate objective：限制新旧策略更新幅度，提升训练稳定性。

PPO 的核心目标可写为：

```text
L_clip(theta) =
E[min(r_t(theta) A_t,
      clip(r_t(theta), 1-epsilon, 1+epsilon) A_t)]
```

其中：

```text
r_t(theta) = pi_theta(a_t | o_t) / pi_theta_old(a_t | o_t)
```

本项目配置中使用并行环境采样、MLP actor-critic、`gamma = 0.99`、`lambda = 0.95`、`clip_param = 0.2`。大量并行环境能够提高样本吞吐，使策略在多种初始状态和动力学扰动下学习更鲁棒的控制行为。

## 10. 轨迹生成与泛化评估

轨迹跟踪任务将目标轨迹作为时间相关命令。训练阶段采样平滑随机轨迹，包括：

- 椭圆/圆轨迹。
- Lissajous/figure-eight 轨迹。
- 单轴正弦轨迹。

这些轨迹提供不同方向、频率、幅值和相位的激励，使策略学习一般性的运动跟踪能力。

评估阶段引入训练分布之外的轨迹：

- Helix：耦合水平圆周和竖直正弦运动，测试三维耦合跟踪。
- Spiral：半径随时间变化，测试幅值变化下的适应能力。
- Chirp：频率随时间上升，测试速度/加速度外推能力。
- Racetrack：直线与半圆组合，测试转弯与直线段切换。

这种训练/评估划分体现了泛化测试思想：策略不能只记住训练轨迹，而应学习基于目标位置误差和目标速度的反馈控制规律。

## 11. 随机化与鲁棒性

为缩小仿真和真实系统之间的差距，本项目使用了多种随机化机制：

- 初始位置随机化。
- 初始线速度和角速度随机化。
- 浮心相对质心偏移随机化。
- 排水体积随机化。
- 轨迹参数随机化。

这些随机化可以视为 domain randomization。训练时让策略接触一族动力学模型和初始条件，目标是学习对模型误差不敏感的反馈策略。理论上，这相当于优化扰动分布下的期望回报：

```text
max_theta E_{xi ~ p(xi)} [J(theta; xi)]
```

其中 `xi` 表示质量、体积、浮心位置、初始状态等环境参数。

仓库中还提供了带偏置的非对称噪声模型，可用于进一步模拟传感器噪声、估计偏差或执行器不确定性。

## 12. 评价指标

该工作可从控制精度、稳定性和能耗三个角度评价：

- 位置误差：`||p_target - p||`，常用 RMSE 汇总。
- 姿态误差：四元数误差或等价旋转角误差。
- 速度误差：轨迹跟踪中比较 `v_target` 与 `v_robot`。
- 动作范数：衡量控制能耗和控制平滑性。
- 越界率/终止率：衡量策略稳定性和安全性。
- 累计奖励：训练阶段的总体优化指标。

其中轨迹跟踪实验更适合使用位置 RMSE、速度 RMSE 和动作范数进行横向比较；定点保持实验更关注收敛时间、稳态位置误差、姿态误差和动作代价。

## 14. 可写入论文/报告的核心表述

本工作的理论基础可以概括为：

> 面向推进器驱动 AUV 的六自由度控制问题，本文基于 Fossen 风格的水下机器人动力学建立仿真模型，在 Isaac/PhysX 刚体动力学基础上补充浮力、相对水流阻尼和附加质量相关流体力。推进器输入经由死区、非线性转速映射、一阶执行器动态和二次推力模型转换为体坐标系下的合力与合力矩。控制问题被建模为连续动作马尔可夫决策过程，策略网络以目标姿态、体坐标误差、目标速度和机器人速度为观测，输出六个推进器命令。通过 PPO 最大化由位置跟踪、姿态跟踪、速度匹配、角速度抑制和动作代价组成的奖励函数，使策略在并行仿真和域随机化条件下学习鲁棒的闭环控制律。训练阶段使用随机平滑轨迹进行策略学习，评估阶段使用分布外轨迹检验策略的泛化能力。

## 15. 需要明确的模型假设

当前实现中有几个假设在论文或报告中应明确说明：

- 刚体惯性、重力和积分由 PhysX 处理，水动力模块只添加外部流体力，避免重复建模。
- 水动力阻尼使用体坐标系对角线性/二次阻尼近似，忽略了完整耦合阻尼项。
- 当前附加质量参数默认为 0，代码结构支持后续加入非零附加质量。
- 观测使用仿真中的完整状态；真实部署时需要状态估计器提供位置、姿态和速度。
- 奖励函数是工程化 shaped reward，并非严格的 Lyapunov 控制律证明。
- 域随机化覆盖的参数范围决定了策略鲁棒性的适用范围，真实系统迁移仍需要实验校准。
